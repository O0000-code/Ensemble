## 二、产品定位回顾

### 2.1 产品是什么
**Ensemble** - macOS 桌面应用，用于管理 Claude Code 的配置：
- Skills（技能）
- MCP Servers（MCP 服务器）
- CLAUDE.md（配置文件）

### 2.2 核心价值主张
- **问题**: Claude Code 每安装一个 MCP/Skill 都占用上下文
- **解决方案**: 场景化配置管理，按项目需求启用
- **独特卖点**: "三位一体"管理 + Scenes 场景预设

### 2.3 目标用户
- Claude Code 用户
- AI 辅助编程爱好者
- 开发者工具爱好者
- macOS 用户

## 四、文案风格版本

### 4.1 夸张宣传版 (Bold/Promotional)
- 强调独特性和创新性
- 使用有力的动词和形容词
- 制造紧迫感和兴奋感

### 4.2 客观陈述版 (Objective/Informative)
- 以事实为基础
- 清晰说明功能和用途
- 适合技术社区

### 4.3 谦虚克制版 (Humble/Subtle)
- 低调展示
- 强调解决个人痛点
- 邀请反馈和建议

## 五、执行阶段规划

### Phase 1: 平台研究
- 各平台特点、用户画像、内容风格
- 成功案例分析
- 最佳发布时间

### Phase 2: 策略制定
- 各平台目标设定
- 内容策略
- 互动策略

### Phase 3: 文案创作
- 各平台 × 各风格版本的文案
- 配图建议
- 链接和 CTA 设计

### Phase 4: 汇总输出
- 完整的宣传手册
- 发布检查清单
- 效果追踪建议

## 六、质量标准

1. **平台适配性**: 内容风格必须符合各平台特点
2. **目标明确**: 每个平台的目标清晰可衡量
3. **可执行性**: 文案可直接使用或略作修改
4. **完整性**: 包含所有必要元素（文案、配图建议、链接等）
5. **多样性**: 提供多种风格选择
