### 1.1 帖子版本 A - 客观分享版

**推荐节点**: /go/share（分享创造）或 /go/create

**标题候选**:
1. `[开源] 做了个管理 Claude Code 配置的 macOS 工具`
2. `分享一个解决 Claude Code MCP 上下文占用问题的工具`
3. `Ensemble - 让 Claude Code 的 Skills/MCP/CLAUDE.md 管理更轻松`

**正文**:

```
大家好，分享一个自己开发的开源工具。

## 背景

用 Claude Code 有一段时间了，逐渐发现一个问题：每安装一个 MCP Server 或 Skill，都会在每次会话中占用额外的上下文。项目多了、工具多了之后，上下文消耗越来越大，有时候还没开始干活，Token 就已经吃掉一大半了。

手动管理配置文件又很麻烦 —— 三个配置分散在不同地方（Skills、MCP Servers、CLAUDE.md），每次切换项目都要手动编辑。

于是花了些时间做了这个工具来解决这个问题。

## Ensemble 是什么

一句话：Claude Code 三位一体配置管理器。

核心思路是「场景化配置管理」—— 不是所有工具都需要时刻启用，而是按项目/场景需求来选择性启用。

## 主要功能

- **Skills 管理**：扫描、分类、标签管理，支持 AI 自动分类
- **MCP Servers 管理**：可视化配置，工具发现
- **CLAUDE.md 管理**：全局设置编辑，一键分发到项目
- **Scenes（场景预设）**：把 Skills + MCPs + CLAUDE.md 组合成预设，一键切换
- **Projects 关联**：项目绑定场景，打开项目自动应用配置
- **Finder 集成**：右键菜单快速操作

## 技术栈

Tauri 2.0 + React + Rust，原生性能，资源占用低。

## 其他信息

- 平台：仅 macOS（暂时）
- 许可证：MIT 开源
- GitHub：[链接]

目前还在早期阶段，功能可能还有不完善的地方。欢迎大家试用，有任何问题或建议都欢迎反馈。

如果觉得有用，给个 star 支持一下 :)
```

### 1.2 帖子版本 B - 技术深度版

**推荐节点**: /go/macos

**标题候选**:
1. `用 Tauri 2.0 做了个 Claude Code 配置管理工具`
2. `[macOS] Ensemble - 基于 Tauri 2.0 的 Claude Code 配置管理器`
3. `Tauri 2.0 + Rust 实践：一个 Claude Code 工具链管理应用`

**正文**:

```
最近用 Tauri 2.0 做了一个 macOS 应用，分享一下。

## 项目背景

Claude Code 的配置管理一直是个痛点：
- Skills 配置在一个地方
- MCP Servers 配置在另一个地方
- CLAUDE.md 又是单独的
- 每个启用的工具都会占用会话上下文

之前一直手动管理 JSON 配置文件，切换项目时经常出错。于是决定做个工具来统一管理。

## 为什么选 Tauri 2.0

1. **原生性能**：Rust 后端，内存占用低
2. **包体积小**：相比 Electron 小很多
3. **macOS 集成好**：Finder 扩展、系统 API 调用方便
4. **2.0 新特性**：多窗口支持、更好的权限系统

## 核心架构

```
Frontend: React + TypeScript
Backend: Rust (Tauri)
Storage: 本地 JSON 配置
```

### 几个技术点

1. **配置热重载**：文件监听 + 增量更新
2. **AI 分类**：调用 Claude API 自动给 Skills 分类打标签
3. **Finder 扩展**：右键菜单集成，用 Swift 写的 Action Extension
4. **场景预设**：类似 Profile 的概念，一键切换整套配置

## 目前实现的功能

- Skills 扫描、分类、批量管理
- MCP Servers 配置编辑、工具发现
- CLAUDE.md 编辑器、项目分发
- Scenes 场景预设管理
- Projects 项目关联
- Finder 右键菜单

## 开源地址

GitHub: [链接]

MIT 许可证，代码完全开源。

---

技术栈选型和实现过程中踩了不少坑，如果大家感兴趣，后续可以单独写一篇技术文章分享。

欢迎试用和反馈，PR 也欢迎。
```
